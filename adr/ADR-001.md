# ADR-001: Core Project Mission

- Status: Accepted

- Date: 2025-11-13

- Title: Guiding Principles: Farmer Autonomy and Shared Value

## Context

Project Gaia consists of two distinct parts: autonomous Farm Nodes (local) and a central Co-op Hub (shared). A clear, non-negotiable philosophy is required to guide all technical and business logic decisions. We must define the fundamental relationship between the farmer and the cooperative to prevent architectural conflicts.

## Decision

We will adopt two core principles as the primary guide for all development:

1. Farmer Autonomy: The Farm Node must be 100% operational for all local tasks (e.g., irrigation, local device rules) even if its connection to the Hub is unavailable for an extended period. The Hub provides enhancements and services, not life support.

2. Shared Value & Explicit Trust: The Hub's value is derived from cooperation. To build the trust this requires, all data sharing from a Farm Node to the Hub must be opt-in by default. The default policy for any new member is share_nothing.

## Consequences

- Positive: This model builds high resilience, as a farm's core operations are not dependent on internet connectivity. The "trust-first" approach is critical for the adoption and success of a cooperative model.

- Positive: This clarifies the architecture: Hub APIs must be asynchronous, and the Farm Node must have a robust "offline-first" mode for syncing.

- Negative: This introduces complexity. We cannot assume the Hub is reachable, so the Farm Node requires more sophisticated local logic and data synchronization capabilities than a simple thin client.
