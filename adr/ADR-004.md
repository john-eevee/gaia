# ADR-004: Monorepo Structure with Shared Build Configuration

- Status: Accepted

- Date: 2025-11-19

- Title: Monorepo Structure with Shared Build Configuration

## Context

The Gaia project consists of multiple Elixir applications (`bouncer`, `hub`,
`testing_facility`) that must share common dependencies and configurations while
maintaining independent deployment capabilities. Without a unified structure, we
face duplication of configuration, inconsistent tooling, and inefficient CI/CD
workflows.

## Decision

We will organize all applications under a `src/` directory in a monorepo
structure with shared build configuration via `Gaia.Build.Mix`:

- All applications reside in `src/{app_name}/` with standard Mix project structure
- A `src/build/mix.exs` file provides `Gaia.Build.Mix.apply/1` that injects
common dependencies, compiler options, and CI aliases into each application's `mix.exs`
- Root-level `.formatter.exs` and `.credo.exs` provide cascading configuration
for all applications
- Each application has its own GitHub Actions workflow with path-based triggers
 (e.g., `src/bouncer/**`)
- The `testing_facility` application provides shared test utilities available
 as a path dependency

Applications integrate by requiring the shared build file and calling `Gaia.Build.Mix.apply/1`:

```elixir
Code.require_file("../build/mix.exs", __DIR__)

def project do
  [...] |> Gaia.Build.Mix.apply()
end
```

## Consequences

- Positive: All applications share consistent tooling (formatting, linting, testing, auditing) with zero configuration duplication. Common dependencies are defined once and applied uniformly.

- Positive: Applications remain independently deployable with their own versions and `mix.lock` files. CI runs selectively based on which application changed.

- Negative: Shared dependencies must be compatible across all applications. Changes to `Gaia.Build.Mix` affect every application simultaneously.

- Negative: New developers must understand the monorepo structure and that most `mix` commands should be run from individual application directories, not the root.
