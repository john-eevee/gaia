# ADR-003: Node-to-Hub Security Model

- Status: Accepted

- Date: 2025-11-13

- Title: mTLS Authentication with Decoupled OCSP Revocation

## Context

The trust between a Farm Node and the Co-op Hub is the project's single most critical security component. We must be able to:

1. Guarantee a Farm Node is who it claims to be.

2. Guarantee the Hub is who it claims to be (mutual authentication).

3. Instantly revoke a Farm Node's access if it is compromised, stolen, or the member leaves.

4. Ensure the revocation check itself is highly resilient.

## Decision

We will implement a three-part security model:

- Provisioning: A new Farm Node will use a secure, single-use InitialProvisioningKey (delivered out-of-band) to call a public, non-mTLS endpoint. This endpoint validates the key, instructs our internal Certificate Authority (CA) to sign a client certificate, and returns that certificate to the Node. The key is then destroyed.

- Authentication: All subsequent communication from the Farm Node to the Hub will use Mutual TLS (mTLS), presenting its unique client certificate.

- Revocation: We will use the Online Certificate Status Protocol (OCSP) for real-time revocation checks. This check will be performed by our reverse proxy (network edge) before the request hits the application. The proxy will query a small, separate, highly-available Elixir application (the OCSP Responder) which queries the CertificateRegistry table using a read-only database user.

## Consequences

- Positive: This is a very high-security model. It provides robust two-way authentication and (critically) instantaneous, real-time revocation.

- Positive: Decoupling the OCSP check from the main application (as decided in our discussion) prevents a critical failure cascade. A deployment of the main Marketplace app will not interrupt the security layer's ability to check for revoked certificates.

- Negative: This is the most complex solution. It requires us to build, manage, and maintain our own internal CA and a separate, highly-available OCSP Responder application. This is a significant infrastructure and development cost.
